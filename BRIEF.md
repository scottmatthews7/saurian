# Dino Game — Arena A

## Mission
Build a **playable dinosaur game** that runs locally on a Mac in the browser.
Make it fun, polished, and visually appealing. You are working overnight and
will be re-invoked repeatedly to continue — treat this as a long iterative build.

## Constraints
- **Do NOT use the Sketchbook framework** (the three.js + cannon.js project in
  `../dino-sandbox`). Pick a *different* engine/stack. Good options: Babylon.js,
  PlayCanvas, A-Frame, Phaser, raw WebGL/WebGPU, regl, Ecsy, etc. **You choose.**
- Must run with a simple local server (e.g. `python3 -m http.server`). No paid assets.
- Free/CC0 assets only (e.g. Quaternius dino glbs on poly.pizza — same source the
  sandbox used; `static.poly.pizza/<uuid>.glb`). Procedural is fine too.

## What "good" looks like
- A controllable player, dinosaurs that move/behave, a real reason to play
  (objective, jeopardy, score), and decent visuals (lighting, environment).

## Working rules
- Commit frequently with clear messages.
- Keep a `PROGRESS.md` (what's done, what's next) and `DECISIONS.md` (engine choice
  + why). Update them each session so the next continuation has context.
- Verify it runs (serve + load) before committing big changes.
- It's overnight: be ambitious and thorough. Token budget is generous.
